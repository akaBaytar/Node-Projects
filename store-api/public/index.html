API Documentation

Overview

This API endpoint allows users to retrieve a list of products based on various filtering, sorting, and pagination options.

Endpoint

GET /api/v1/products

Query Parameters

- featured: Filter products by featured status. Accepted values: true or false
- company: Filter products by company name
- name: Filter products by name (case-insensitive, supports partial matches)
- sort: Sort products by specific parameters. Multiple parameters can be separated by commas
- select: Select specific fields in the response object. Multiple fields can be selected by separating them with commas
- filter: Filter products based on numerical criteria. Supports comparison operators < , > , <= , >= , and =
Multiple filters can be applied by separating them with commas. Filters are applied on fields price and rating

Sample Usage

GET /api/v1/products

Retrieves all products.

GET /api/v1/products?featured=true

Retrieves featured products.

GET /api/v1/products?company=Acme

Retrieves products from the company "Acme".

GET /api/v1/products?name=widget

Retrieves products with the word "widget" in their name (case-insensitive).

GET /api/v1/products?sort=price,-rating

Sorts products by price in ascending order and rating in descending order.

GET /api/v1/products?select=name,price,rating

Selects only the name, price, and rating fields in the response.

GET /api/v1/products?filter=price<=100,rating>4

Filters products with a price less than or equal to 100 and a rating greater than 4.

GET /api/v1/products?page=1&limit=10

Retrieves the first page of products with a limit of 10 products per page.

Response

The response consists of the following key components:

- page: The current page number of the results.
- limit: The maximum number of products returned per page.
- products: An array containing product objects that match the specified criteria.

Sample Response

{
  "page": 1,
  "limit": 10,
  "products": [
    {
      "name": "Widget A",
      "price": 50,
      "rating": 4.5
    },
    {
      "name": "Widget B",
      "price": 75,
      "rating": 4.8
    },
    // Additional product objects...
  ]
}

Notes

  • If no query parameters are provided, the endpoint returns all products.
  • Pagination is applied by default with a limit of 12 products per page and defaults to page 1 if not specified.
